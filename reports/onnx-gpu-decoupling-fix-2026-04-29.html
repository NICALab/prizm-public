Inputs and Parameters

  • User request: update this repo copy instead of the earlier one.
  • Fix scope: ONNX GPU selection in infer.py and corresponding readiness logging in batch_segmentation_core.py.

Outputs

  • ONNX Runtime provider discovery now happens via a dedicated helper with best-effort DLL preloading.
  • ONNX CUDA sessions are attempted whenever CUDAExecutionProvider is available, even if PyTorch reports no CUDA.
  • Fallback device selection now defaults to GPU 0 when ONNX Runtime exposes CUDA but PyTorch cannot enumerate devices.
  • Batch logs now include the ONNX available-provider list.

Errors Encountered

  • No blocking command errors occurred.
  • No built-in test suite was present, so focused mocked validations were used.

Debugging Steps

  • Confirmed this repo copy still had the original if self.device == "cuda" ONNX gate.
  • Patched both the top-level ONNX selection path and the device-ID helper to avoid a second hidden PyTorch dependency.
  • Validated both the Windows-style mismatch case and the CPU-only fallback case.

Fixes Applied

  • Added onnx_available_providers and onnx_requested_providers state in infer.py.
  • Added _get_onnx_available_providers().
  • Decoupled ONNX GPU selection from torch.cuda.is_available().
  • Updated batch log messages to include available ONNX providers.
  • Removed generated __pycache__ folders after validation.
